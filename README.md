# YOLO Feedback

  Report bugs, suggest features, or share feedback.

  ## Reporting Issues

  [Create a new issue](https://github.com/yolocorp-dev/feedback/issues/new) with:

  **Bug reports:**
  - CLI command that failed
  - Error message/output
  - Expected vs actual behavior
  - **Output of `yolo doctor`** (checks CLI/Docker/backend health)

  **Feature requests:**
  - Problem you're trying to solve
  - Proposed solution
  - Use case/context

  **General feedback:**
  - What works well / doesn't
  - UX pain points
  - Documentation gaps

  ## Before Reporting

  1. Run `yolo doctor` to check for common issues
  2. Check [existing issues](https://github.com/yolocorp-dev/feedback/issues)
  3. Try `yolo update` if version is outdated
  4. Verify you're logged in: `yolo whoami`

  ## System Info Template

  $ yolo doctor
  [paste full output here]
